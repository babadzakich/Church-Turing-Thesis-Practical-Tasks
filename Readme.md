# Анализ достигающих определений

## Условие

Дан текст программы на игрушечном языке программирования.
Необходимо:

1. распарсить программу;
2. построить для неё CFG в канонической форме;
3. выполнить анализ достигающих определений;
4. вывести CFG, аннотированный комментариями о том, из каких базовых блоков в данный блок доходят значения переменных.


Язык описан в файле: [toy_lang.md](./toy_lang.md)

Формат CFG описан в файле: [cfg_spec.md](./cfg_spec.md)


## Что требуется вычислить

Для каждого базового блока нужно определить, **какие определения переменных достигают входа в этот блок**.

В этой задаче определение переменной задаётся парой:

```text
(variable, block_id)
```

Это означает, что текущее значение переменной было определено в блоке `block_id`.

Если для переменной в блок не достигает ни одно определение, используется специальное значение:

```text
(variable, undef)
```

Это означает, что в данную точку программы переменная может прийти только как неопределённая.


## Формат вывода

Перед каждым базовым блоком должен стоять комментарий вида:

```text
// (x, 0), (x, 1), (y, 0), (y, 1)
```

Это означает, что во вход данного блока:

* значение `x` может прийти из `block0` или `block1`;
* значение `y` может прийти из `block0` или `block1`.

После комментария печатается сам блок в формате CFG из задачи 1.

Файл с результатом записывается в `<input_path>.ans`

## Правила для `undef`

Если блок не достигает ни одно определение переменной, то для неё нужно вывести:

```text
(x, undef)
```

Например, для входного блока программы все переменные всегда имеют значение `undef`, так как до начала программы ни одного определения ещё не было.


## Порядок вывода в комментарии

Чтобы все корректные решения имели одинаковый вывод, пары в комментарии нужно печатать в следующем порядке:

1. сначала переменные в лексикографическом порядке;
2. для каждой переменной:

   * сначала `(var, undef)`, если он есть;
   * затем блоки по возрастанию номера.

Например:

```text
// (x, undef), (y, 0), (y, 2), (z, 1)
```

## Какие переменные нужно выводить

В комментарии перед каждым блоком должны присутствовать **все переменные программы**, даже если для некоторых из них достигает только `undef`.


## Что считается определением внутри блока

Если в одном блоке переменная определяется несколько раз, то для выхода из блока значимым считается **последнее** определение этой переменной внутри блока.

Например, в блоке

```text
block0
x = 1
x = 2
goto block1;
```

на выходе из `block0` для переменной `x` должно быть только:

```text
(x, 0)
```

## Что означает аннотация блока

Комментарий перед блоком описывает **состояние на входе в блок**, а не на выходе из него.

То есть комментарий перед `block3` показывает, какие определения достигают начала `block3`, до выполнения его инструкций.


## Пример

### Вход

```c
x = 1;
y = 2;
if (x < y) {
    x = 3;
} else {
    y = 4;
}
z = x + y;
print(z > 0);
```

### CFG с аннотацией

```text
// (x, undef), (y, undef), (z, undef)
block0
x = 1
y = 2
if (x < y) goto block1; else goto block2;

// (x, 0), (y, 0), (z, undef)
block1
x = 3
goto block3;

// (x, 0), (y, 0), (z, undef)
block2
y = 4
goto block3;

// (x, 0), (x, 1), (y, 0), (y, 2), (z, undef)
block3
z = x + y
print(z > 0)
return;
```

### Пояснение

* На входе в `block0` все переменные имеют `undef`.
* На входе в `block1` и `block2` значения `x` и `y` приходят из `block0`.
* На входе в `block3`:

  * `x` может прийти либо из `block0`, либо из `block1`;
  * `y` может прийти либо из `block0`, либо из `block2`;
  * `z` ещё нигде не определялся, поэтому `(z, undef)`.


## Требования

* Нужно сначала построить CFG в канонической форме.
* Формат CFG должен совпадать с [cfg_spec.md](./cfg_spec.md).
* Комментарий должен стоять **непосредственно перед** соответствующим блоком.
* Аннотация должна описывать только вход блока.
* Во всех комментариях должны быть перечислены все переменные программы.
* Формат вывода должен точно совпадать со спецификацией.



## Краткая формулировка

Для каждого базового блока необходимо вывести, из каких блоков в него могут прийти последние определения переменных.
Если переменная к началу блока ещё нигде не определялась, нужно вывести `(var, undef)`.
