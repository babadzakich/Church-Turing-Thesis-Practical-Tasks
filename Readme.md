# Причинная согласованность

## Теория

В распределённых системах часто требуется, чтобы события наблюдались в корректном логическом порядке. Если одно событие причинно зависит от другого, то все узлы системы должны наблюдать их согласованно.

Событие A считается причинно предшествующим событию B (обозначается A→B), если:
- A произошло раньше B в рамках одного процесса;
- A — отправка сообщения, а B — его получение;
- существует цепочка таких зависимостей.

Такое отношение называется **happens-before**.

Требование **causal consistency** формулируется так:

> Если событие A→B, то ни один узел не должен наблюдать B раньше A.

---

## Контекст
Было необходимо разработать распределённую систему для редактирования документов в условиях нестабильной сети.

**Сеть:**
- не теряет сообщения;
- может доставлять их с произвольной задержкой;
- может доставлять сообщения в разном порядке разным узлам.


Можно получить доступ с правом редактирования из любого узла системы.
Изначально документ пустой.

У пользователя есть только команда `append <text> `
Данная команда добавляет текст в конец документа и отправляет сообщение `append <text>` на другие узлы. Таким образом каждый узел самостоятельно собирает актуальное состояние.

Если произошли 2 события A и B, такие что не A→B и не B→A, то программа сообщает о конфликте и завершает исполнение.


По итогу работ было разработано следующее приложение: [distributed_editor.c](./distributed_editor.c)


Разработчики понимали, что рассчитывать на стабильность сети не приходится. Поэтому решили использовать логические часы Лэмпорта для определения порядка событий.

Ведь, как известно, если A→B, то время события A < времени события B.

Поэтому разработчики, используя время событий, определяли их отношение happens-before.
