# Машина Тьюринга

## Условие задачи

    В одной IT-компании случился кризис: слёг единственный сервер, на котором хранились все пароли. Админ, дёргая проводки и приговаривая: «Я всё предвидел, кроме этого», в отчаянии вспомнил про пыльный ящик в подсобке.
    В ящике лежала Машина Тьюринга образца 1936 года. На коробке было написано от руки: «Открывать в случае конца света или дедлайна через час».
    Машину подключили к монитору. Она пискнула. Оказалось, старушка умеет только одно — копировать строки из символов a и b, разделяя их символом #. Но именно это и нужно: чтобы восстановить базу, надо каждую запись продублировать.
    Ваша задача — написать эмулятор машины Тьюринга, который берёт строку и превращает её в строку вида:
    S#S
    где S — исходная строка.
    

```
Примеры
"ab" → "ab#ab"
"a" → "a#a"
"" → "#"
```

Если машина справится — пароли спасены. Если нет... возможно, админ вспомнит, где записывал их на салфетке.

## Схема задачи

- участник сдаёт код библиотеки и `Makefile` в своей папке `teams/<team>/`
- Важно использовать только те интерфейсы что есть в `machine_turing.h`
- `machine_turing.c` не обязан содержать `main`
- `make build` собирает объектный файл решения
- `make test TEST_FILE=/abs/path/to/test.c` компилирует и запускает один конкретный тестовый раннер

Контракт для тестов:

- каждый тест хранится отдельным `.c`-файлом
- тест подключает `machine_turing.h`
- тест содержит свой `main`
- успешный тест возвращает `0`
- неуспешный тест возвращает ненулевой код и может писать ошибку в `stderr`
