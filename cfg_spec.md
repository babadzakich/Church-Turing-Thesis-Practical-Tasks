# Формат CFG (Control Flow Graph)

Этот документ описывает **канонический текстовый формат CFG**, который:

* является выходом задачи 1 (парсер + CFG),
* является входом задачи 2 (анализ достигающих определений).

Формат строго фиксирован. Все корректные решения должны выдавать **одинаковый CFG** для одной и той же программы.


# Общий вид

CFG состоит из набора базовых блоков:

```text
block<i>
<instruction>
...
<terminator>
```

Пример:

```text
block0
x = 1
if (x < 0) goto block1; else goto block2;
```

# Инструкции

Допустимые инструкции внутри блока:

### Присваивания

```text
x = 5
x = a + b
x = read()
```

### Вывод

```text
print(true)
print(false)
print(x < y)
```


# Завершающая инструкция блока (terminator)

Каждый блок **обязательно заканчивается ровно одной** из инструкций:

### Условный переход

```text
if (cond) goto blockX; else goto blockY;
```

### Безусловный переход

```text
goto blockX;
```

### Завершение программы

```text
return;
```


# Правила построения CFG

## 1. Разбиение на блоки

Программа разбивается на базовые блоки **только по операторам `if`**.

### Линейный участок

Максимальная последовательность инструкций без `if` образует один блок.

## 2. Обработка `if`

Оператор:

```c
if (cond) { A } else { B }
```

преобразуется следующим образом:

1. текущий блок заканчивается условием:

   ```text
   if (cond) goto block_then; else goto block_else;
   ```
2. создаётся CFG then-ветки;
3. создаётся CFG else-ветки;
4. создаётся **один общий merge-блок**, куда сходятся обе ветки.


## 3. Merge-блок

После каждого `if` обязательно создаётся блок слияния, который содержит код после `if`.

Обе ветки должны завершаться переходом в этот блок:

```text
goto block_merge;
```


## 4. Завершение программы

* В CFG должен быть **ровно один `return;`**.
* Он находится в **последнем блоке программы**.

## 5. Нумерация блоков

Блоки нумеруются в порядке их создания:

1. текущий блок,
2. then-ветка,
3. else-ветка,
4. merge-блок.

Это правило обязательно.

# Что запрещено

* разбивать блоки по отдельным инструкциям;
* создавать блок после `print` или присваивания; (разбиваем cfg только по if выражениям)
* не создавать merge-блок после `if`;
* дублировать код после `if` в обе ветки;
* иметь несколько `return;`;
* произвольно менять порядок нумерации блоков.

# Пример 1

## Исходная программа

```c
x = 1;
y = 2;
if (x < y) {
    x = 3;
} else {
    y = 4;
}
z = x + y;
print(z > 0);
```

## CFG

```text
block0:
x = 1;
y = 2;
if (x < y) goto block1; else goto block2;

block1:
x = 3;
goto block3;

block2:
y = 4;
goto block3;

block3:
z = x + y;
print(z > 0);
return;
```

# Пример 2 (вложенный if)

## Исходная программа

```c
x = read();
if (x < 0) {
    if (x < -5) {
        y = 1;
    } else {
        y = 2;
    }
} else {
    y = 3;
}
print(y > 0);
```

## CFG

```text
block0:
x = read();
if (x < 0) goto block1; else goto block2;

block1:
if (x < -5) goto block4; else goto block5;

block2:
y = 3;
goto block3;

block3:
print(y > 0);
return;

block4:
y = 1;
goto block6;

block5:
y = 2;
goto block6;

block6:
goto block3;

```


# Пример 3 (без ветвления)

## Исходная программа

```c
x = 5;
y = x + 1;
print(y > 0);
```

## CFG

```text
block0:
x = 5;
y = x + 1;
print(y > 0);
return;
```

# Краткая формулировка

* Разбиваем программу на блоки только по `if`.
* `if` всегда завершает блок.
* У `if` всегда две ветки.
* После `if` всегда один merge-блок.
* В программе ровно один `return;` в конце.
* Блоки нумеруются: текущий → then → else → merge.


Этот формат гарантирует, что все корректные реализации будут выдавать одинаковый CFG.
